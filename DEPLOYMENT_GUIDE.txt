ApplyAndFly — Deployment Guide (Run Without Your Laptop)
==========================================================

GOAL
----
Right now the bot only works while you have `npm run dev` running on your
own machine in PyCharm. To make it "just scan a QR code / open a link,"
the server needs to run on a public host 24/7, with a public HTTPS URL.
A QR code is trivial once you have that URL — the real work is deployment.

IMPORTANT LIMITATION TO KNOW UP FRONT
--------------------------------------
This app currently supports ONE user only (one Gmail account, one WhatsApp
recipient, stored in memory). A QR code / shared link would let anyone open
the /auth/google page, but it would overwrite YOUR stored Gmail auth with
THEIRS. This guide assumes the QR code / link is for YOUR OWN repeated use
(e.g. re-authenticating from your phone), not for other people to use the
same bot. Turning this into a true multi-user product is a separate,
bigger project (needs a database, per-user tokens, etc.).


OPTION A — Render.com (recommended for a first deploy, free tier available)
-----------------------------------------------------------------------------
1. Push this project to a GitHub repository (private is fine).
   - If not already a git repo: `git init`, `git add .`, `git commit -m "init"`
   - IMPORTANT: add a `.gitignore` with `node_modules/` and `.env` in it
     BEFORE committing, so your secrets never get pushed to GitHub.
   - Create a new repo on GitHub, then `git remote add origin <url>` and
     `git push -u origin main`.

2. Go to https://render.com and sign up (free, no card required for the
   free tier).

3. Click "New +" -> "Web Service" -> connect your GitHub repo.

4. Configure:
   - Runtime: Node
   - Build Command: npm install
   - Start Command: npm start
   - Instance Type: Free

5. Add Environment Variables (Render dashboard -> your service ->
   "Environment" tab). Add every value currently in your local .env:
     GOOGLE_CLIENT_ID
     GOOGLE_CLIENT_SECRET
     GOOGLE_REDIRECT_URI   <- set this to the NEW Render URL, see step 7
     WHATSAPP_TOKEN
     WHATSAPP_PHONE_NUMBER_ID
     WHATSAPP_TO_NUMBER
     GROQ_API_KEY
     PORT (Render sets its own PORT automatically; you can leave this out,
           the code already falls back to Render's assigned port)

6. Deploy. Render will give you a public URL like:
     https://applyandfly.onrender.com

7. Update GOOGLE_REDIRECT_URI:
   - In Render's environment variables, set it to:
     https://applyandfly.onrender.com/auth/google/callback
   - In Google Cloud Console (console.cloud.google.com) -> APIs & Services
     -> Credentials -> your OAuth Client -> "Authorized redirect URIs" ->
     add that same URL. Save.
   - Redeploy the Render service so it picks up the new env variable.

8. Test: open https://applyandfly.onrender.com/auth/google in a browser,
   complete Google login, confirm you see "Auth successful."

9. Generate a QR code pointing to that URL:
   - Easiest: use a free QR generator like https://www.qr-code-generator.com
     (paste in your /auth/google URL, download the PNG).
   - Or, if you want it generated by code later, the `qrcode` npm package
     can do this in two lines — ask me to wire this in when you're ready.

RENDER FREE TIER CAVEAT (important):
   Render's free web services "spin down" after ~15 minutes with no
   incoming HTTP requests, and take ~30-60 seconds to "wake up" on the
   next request. Since this bot's email-checking relies on an internal
   timer (setInterval) running INSIDE the process, that timer stops
   whenever Render puts the service to sleep — meaning it will NOT check
   your email every 60 seconds around the clock on the free tier, only
   while the service happens to be awake.
   Fixes if you need true 24/7 background checking:
     - Upgrade to Render's paid tier (~$7/month, keeps it always running), OR
     - Use a free "uptime pinger" service (e.g. UptimeRobot, free) that
       hits your URL every 5-10 minutes to keep it awake (not perfectly
       reliable, but works for most personal use), OR
     - Use a host with a true always-on free tier, see Option B.


OPTION B — Fly.io (always-on free allowance, slightly more setup)
--------------------------------------------------------------------
1. Install the flyctl CLI (instructions at https://fly.io/docs/flyctl/install)
2. `fly auth signup` (free, requires a card on file for verification but
   the free allowance covers a small always-on app with no charge)
3. In the project folder: `fly launch` — it detects Node.js and generates
   a fly.toml config. Choose a unique app name (this becomes your subdomain,
   e.g. applyandfly.fly.dev).
4. Set secrets instead of a .env file:
     fly secrets set GOOGLE_CLIENT_ID=... GOOGLE_CLIENT_SECRET=...
       GOOGLE_REDIRECT_URI=https://applyandfly.fly.dev/auth/google/callback
       WHATSAPP_TOKEN=... WHATSAPP_PHONE_NUMBER_ID=...
       WHATSAPP_TO_NUMBER=... GROQ_API_KEY=...
5. `fly deploy`
6. Same as Render: add the callback URL to Google Cloud Console's
   Authorized redirect URIs.
7. Fly's free allowance (as of writing) covers small always-on machines,
   so this avoids Render's sleep/wake issue — but check Fly's current
   pricing page since free-tier terms change over time.


OPTION C — Railway.app
-------------------------
Similar flow to Render: connect GitHub repo, set environment variables in
the dashboard, deploy, get a public URL, update the Google redirect URI.
Railway's free tier is credit-based (a small monthly free credit) rather
than a permanent free plan — fine for light personal use, but track usage
so you don't get an unexpected bill once the credit runs out.


RECOMMENDATION
--------------
Start with Render (Option A) since it's the simplest to set up and fully
free with no card required. If you find the "sleeps after 15 min" behavior
annoying for a bot that's supposed to run in the background, either add a
free UptimeRobot ping, or move to Fly.io for a true always-on free machine.
